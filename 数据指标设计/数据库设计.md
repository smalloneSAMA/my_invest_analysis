# A股公司分析系统 - PostgreSQL 数据库设计方案

## 一、设计概述

### 1.1 设计原则

- **宽表优先**：三大财报（资产负债表/利润表/现金流量表）采用宽表设计，每个财报项目对应一个独立列
- **物化存储衍生指标**：ROE/PE/PB等组合指标预计算落表，日频刷新
- **多源分级**：2-3个数据源，按数据类别设置优先级，主源缺数据自动降级
- **NULL安全**：财务数据默认NULL，0仅表示"真正的零"，业务层禁止COALESCE(N, 0)

### 1.2 技术选型

| 项目     | 选择                           |
| -------- | ------------------------------ |
| 数据库   | PostgreSQL 16.14               |
| 字符集   | UTF8                           |
| 排序规则 | zh_CN.utf8                     |
| 分区策略 | daily_quotes 按年份 RANGE 分区 |
| 物化刷新 | pg_cron 定时 + 财报事件触发    |

## 二、坑1 解决方案：除权除息 & 历史复权

### 2.1 设计思路

 采用**复权因子法**（行业标准，TuShare/Baostock/Wind通用）：

- `daily_quotes` 表存储 **原始OHLCV** + **日级前复权因子（pre_adj_factor）**
- `adjustment_events` 表记录每次除权除息事件（送股/转增/配股/现金分红）
- 最新交易日复权因子 = 1.0，历史逐日前推
- 任意历史日的前复权价格 = `raw_close * pre_adj_factor`

### 2.2 核心表结构

```sql
 -- 除权除息事件表（审计追溯）                                                                                                                               
 CREATE TABLE adjustment_events (                                                                                                                            
     id              BIGSERIAL PRIMARY KEY,                                                                                                                  
     company_id      INTEGER NOT NULL REFERENCES companies(id),                                                                                              
     ex_date         DATE NOT NULL,           -- 除权除息日                                                                                                  
     event_type      VARCHAR(20) NOT NULL,    -- 'dividend_cash','dividend_stock','split','rights_issue'                                                     
     cash_per_share  NUMERIC(12,6),           -- 每股现金分红                                                                                                
     bonus_ratio     NUMERIC(10,6),           -- 送股比例（10送N → N/10）                                                                                    
     transfer_ratio  NUMERIC(10,6),           -- 转增比例                                                                                                    
     rights_ratio    NUMERIC(10,6),           -- 配股比例                                                                                                    
     rights_price    NUMERIC(12,4),           -- 配股价                                                                                                      
     adj_ratio       NUMERIC(16,12) NOT NULL, -- 本次调整系数                                                                                                
     source_id       SMALLINT REFERENCES data_sources(id),                                                                                                   
     created_at      TIMESTAMPTZ DEFAULT now()                                                                                                               
 );
```

```sql
 -- 日行情表（关键字段：pre_adj_factor）                                                                                                                     
 CREATE TABLE daily_quotes (                                                                                                                                 
     id              BIGSERIAL,                                                                                                                              
     company_id      INTEGER NOT NULL,                                                                                                                       
     trade_date      DATE NOT NULL,                                                                                                                          
     open            NUMERIC(12,4),           -- 原始开盘价                                                                                                  
     high            NUMERIC(12,4),           -- 原始最高价                                                                                                  
     low             NUMERIC(12,4),           -- 原始最低价                                                                                                  
     close           NUMERIC(12,4),           -- 原始收盘价                                                                                                  
     pre_close       NUMERIC(12,4),           -- 原始前收盘价                                                                                                
     volume          NUMERIC(20,0),           -- 成交量（股）                                                                                                
     amount          NUMERIC(20,2),           -- 成交金额（元）                                                                                              
     pre_adj_factor  NUMERIC(16,12) NOT NULL DEFAULT 1.0, -- ★ 前复权因子                                                                                    
     -- ... 其他字段                                                                                                                                         
 );
```

### 2.3 复权计算流程

1. 数据采集时存储原始价格
2. `adjustment_events` 记录每次除权事件及调整系数
3. 每日批量更新 `daily_quotes.pre_adj_factor`（从最新日=1.0逐日向前推算）
4. 查询视图 `v_daily_quotes_adj` 实时输出前复权价

```sql
 -- 前复权视图（示例）                                                                                                                                       
 CREATE VIEW v_daily_quotes_adj AS                                                                                                                           
 SELECT                                                                                                                                                      
     company_id, trade_date,                                                                                                                                 
     ROUND(open  * pre_adj_factor, 4)   AS adj_open,                                                                                                         
     ROUND(high  * pre_adj_factor, 4)   AS adj_high,                                                                                                         
     ROUND(low   * pre_adj_factor, 4)   AS adj_low,                                                                                                          
     ROUND(close * pre_adj_factor, 4)   AS adj_close,                                                                                                        
     volume, amount, pre_adj_factor                                                                                                                          
 FROM daily_quotes;
```

### 2.4 为什么不用"仅存原始价 + 事后计算"

- 事后计算需要每次查询时关联 events 表做累积乘算，复杂度O(n×m)，不可接受
- 复权因子法将复杂度前置到写入时段，查询时只需一次乘法，O(1)

## 三、坑2 解决方案：多数据源冲突

### 3.1 设计思路

 每个数据表增加 `source_id` + `data_fetch_time` 双字段，配合**数据源优先级表**实现分级策略。

### 3.2 核心结构

```sql
 -- 数据源注册表                                                                                                                                             
 CREATE TABLE data_sources (                                                                                                                                 
     id          SMALLSERIAL PRIMARY KEY,                                                                                                                    
     source_code VARCHAR(20) UNIQUE NOT NULL, -- 'mootdx','tencent','eastmoney','tushare'                                                                    
     source_name VARCHAR(100) NOT NULL,                                                                                                                      
     base_url    VARCHAR(255),                                                                                                                               
     is_active   BOOLEAN DEFAULT true,                                                                                                                       
     rate_limit  INTEGER,                     -- 每分钟请求上限                                                                                              
     created_at  TIMESTAMPTZ DEFAULT now()                                                                                                                   
 );                                                                                                                                                          
                                                                                                                                                             
 -- 数据源优先级表（按数据类别定制）                                                                                                                         
 CREATE TABLE data_source_priority (                                                                                                                         
     id              SMALLSERIAL PRIMARY KEY,                                                                                                                
     data_category   VARCHAR(50) NOT NULL,    -- 'daily_quote','balance_sheet','income_statement','dividend','share_capital'                                 
     source_id       SMALLINT NOT NULL REFERENCES data_sources(id),                                                                                          
     priority        SMALLINT NOT NULL,       -- 1最高优先级                                                                                                 
     is_fallback     BOOLEAN DEFAULT false,   -- 是否仅作备胎                                                                                                
     UNIQUE(data_category, source_id),                                                                                                                       
     UNIQUE(data_category, priority)          -- 同类下优先级不重复                                                                                          
 );
```

### 3.3 数据表中的多源约束

```sql
 -- 示例：日行情表的多源设计                                                                                                                                 
 CREATE TABLE daily_quotes (                                                                                                                                 
     -- ... 数据字段 ...                                                                                                                                     
     source_id       SMALLINT NOT NULL REFERENCES data_sources(id),                                                                                          
     data_fetch_time TIMESTAMPTZ NOT NULL DEFAULT now(),                                                                                                     
     data_version    INTEGER DEFAULT 1,       -- 同源数据版本号（重抓后递增）                                                                                
     -- ★ 关键约束：同一公司+日期+数据源不重复                                                                                                               
     CONSTRAINT uq_daily_quote UNIQUE (company_id, trade_date, source_id)                                                                                    
 );
```

### 3.4 业务层取数策略

```sql
 -- 优先级取数函数（示例）                                                                                                                                   
 CREATE FUNCTION get_best_daily_quote(p_company_id INTEGER, p_trade_date DATE)                                                                               
 RETURNS TABLE(...) AS $$                                                                                                                                    
 BEGIN                                                                                                                                                       
     RETURN QUERY                                                                                                                                            
     SELECT dq.*                                                                                                                                             
     FROM daily_quotes dq                                                                                                                                    
     JOIN data_source_priority dsp                                                                                                                           
         ON dsp.data_category = 'daily_quote'                                                                                                                
         AND dsp.source_id = dq.source_id                                                                                                                    
     WHERE dq.company_id = p_company_id                                                                                                                      
       AND dq.trade_date = p_trade_date                                                                                                                      
       AND dsp.priority > 0                                                                                                                                  
     ORDER BY dsp.priority ASC                                                                                                                               
     LIMIT 1;  -- 取最高优先级的第一条                                                                                                                       
 END;                                                                                                                                                        
 $$ LANGUAGE plpgsql STABLE;
```

### 3.5 冲突检测机制

- 写入时 `ON CONFLICT (company_id, date, source_id) DO UPDATE` 确保同源不重复
- 定时任务对比不同源的同日数据，差异>阈值写入 `data_discrepancy_log` 告警表
- 数据质量标记字段 `data_quality_flag`：'verified'/'discrepant'/'estimated'

## 四、坑3 解决方案：NULL 与 0 语义混淆

### 4.1 设计原则

| 场景           | 存储值   | 说明                     |
| -------------- | -------- | ------------------------ |
| 真实为零       | `0`    | 如某季度无投资活动现金流 |
| 数据缺失       | `NULL` | 如未披露、数据源不提供   |
| 不适用         | `NULL` | 如银行业不适用存货指标   |
| 计算中分母为零 | `NULL` | 衍生指标留NULL，不填0    |

### 4.2 字段级约束

```sql
 -- 资产负债表（片段）                                                                                                                                       
 CREATE TABLE balance_sheet (                                                                                                                                
     -- ...                                                                                                                                                  
     total_assets        NUMERIC(20,4)   DEFAULT NULL,  -- ★ 默认NULL                                                                                        
     total_liabilities   NUMERIC(20,4)   DEFAULT NULL,                                                                                                       
     -- 只有"肯定为零"的字段才允许DEFAULT 0                                                                                                                  
     goodwill            NUMERIC(20,4)   DEFAULT NULL,  -- 商誉：无商誉公司留NULL（未取到数据）还是0（确认无商誉）？                                         
     -- → 答案是：数据采集阶段留NULL，业务确认后更新为0                                                                                                      
     -- ...                                                                                                                                                  
     -- ★ 元数据区分NULL含义                                                                                                                                 
     data_completeness   VARCHAR(20) DEFAULT 'partial', -- 'full'/'partial'/'estimated'                                                                      
     data_quality_flag   VARCHAR(20) DEFAULT NULL,      -- 'verified'/'discrepant'/'estimated'                                                               
     -- COMMENT ON COLUMN 明确语义                                                                                                                           
 );                                                                                                                                                          
                                                                                                                                                             
 COMMENT ON COLUMN balance_sheet.total_assets IS '总资产（元）。NULL表示数据未获取，0仅当公司确实无资产';                                                    
 COMMENT ON COLUMN balance_sheet.goodwill IS '商誉（元）。NULL=数据缺失，0=确认无商誉';                                                                      
```

### 4.3 衍生指标计算的安全模式

```sql
 -- ROE物化视图中的安全计算示例                                                                                                                              
 CREATE MATERIALIZED VIEW mv_roe AS                                                                                                                          
 SELECT                                                                                                                                                      
     bs.company_id, bs.report_date,                                                                                                                          
     CASE                                                                                                                                                    
         WHEN is_.parent_np IS NULL OR bs.equity_parent IS NULL                                                                                              
             THEN NULL          -- ★ 任一输入为NULL，结果为NULL                                                                                              
         WHEN bs.equity_parent = 0                                                                                                                           
             THEN NULL          -- ★ 分母为0，结果为NULL（不填0误导）                                                                                        
         ELSE ROUND(is_.parent_np / bs.equity_parent * 100, 2)                                                                                               
     END AS roe                                                                                                                                              
 FROM balance_sheet bs                                                                                                                                       
 JOIN income_statement is_ USING (company_id, report_date);                                                                                                  
```

### 4.4 业务层约定（写入代码规范）

```python
 # Python业务层示例：禁止的写法                                                                                                                              
 roe = net_profit / equity if equity else 0  # ❌ 分母缺失直接填0                                                                                            
                                                                                                                                                             
 # 正确写法                                                                                                                                                  
 def safe_ratio(numerator, denominator):                                                                                                                     
     if numerator is None or denominator is None:                                                                                                            
         return None       # 缺失就是缺失                                                                                                                    
     if denominator == 0:                                                                                                                                    
         return None       # 无穷大也是缺失                                                                                                                  
     return numerator / denominator                                                                                                                          
```

```sql
 CREATE FUNCTION get_priority_financial(

     p_report_date DATE,                                                                                                                                     
     p_report_type VARCHAR,

 ) RETURNS SETOF RECORD AS $$                                                                                                                                
 DECLARE                                                                                                                                                     
     sql TEXT;                                                                                                                                               
 BEGIN                                                                                                                                                       
     sql := format('                                                                                                                                         
         SELECT t.* FROM %I t                                                                                                                                
         JOIN data_source_priority dsp

         WHERE t.company_id = $1 AND t.report_date = $2 AND t.report_type = $3                                                                               
         ORDER BY dsp.priority ASC LIMIT 1                                                                                                                   
     ', p_table_name, p_table_name);                                                                                                                         
     RETURN QUERY EXECUTE sql USING p_company_id, p_report_date, p_report_type;                                                                              
 END;
```

### 6.2 ROE物化刷新（解决坑3）

```sql
 -- ROE的加权平均计算（年度）                                                                                                                                
 INSERT INTO derived_metrics_financial (company_id, report_date, report_type, roe, calc_time)                                                                
 SELECT                                                                                                                                                      
     bs.company_id, bs.report_date, 'annual',                                                                                                                
     CASE                                                                                                                                                    
         WHEN is_.parent_np IS NULL OR bs_avg.avg_equity IS NULL OR bs_avg.avg_equity = 0                                                                    
         THEN NULL   -- ★ NULL安全                                                                                                                           
         ELSE ROUND(is_.parent_np / bs_avg.avg_equity * 100, 4)                                                                                              
     END,                                                                                                                                                    
     now()                                                                                                                                                   
 FROM income_statement is_                                                                                                                                   
 JOIN (                                                                                                                                                      
     SELECT company_id, report_date,                                                                                                                         
            (equity_parent + LAG(equity_parent) OVER w) / 2.0 AS avg_equity                                                                                  
     FROM balance_sheet                                                                                                                                      
     WINDOW w AS (PARTITION BY company_id ORDER BY report_date)                                                                                              
 ) bs_avg USING (company_id, report_date)                                                                                                                    
 WHERE is_.report_type = 'annual'                                                                                                                            
 ON CONFLICT (company_id, report_date, report_type) DO UPDATE SET                                                                                            
     roe = EXCLUDED.roe,                                                                                                                                     
     calc_time = now();
```
